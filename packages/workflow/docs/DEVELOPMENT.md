# Workflow Development Guide

## Daily Workflow
- Implement changes against package contracts first.
- Run build/tests after behavior changes.
- Keep changed files lint/type-clean.

## Safety Checklist
- Preserve path-only linking rules.
- Keep event ownership/prefix rules strict.
- Avoid hardcoded event strings and fallback logic.

## Documentation Rule
- Keep package docs current and concise.
- Move historical planning logs to task/history documents.
