| 60  | Mocking the very collaborator whose contract holds the bug; no cold-state test for a live seam                                                                      | A "live seam" (a runtime op invoked between session start and the first turn — preset/model/permission re-apply, etc.) MUST have at least one integration test exercising the **real** collaborator in its **cold** (not-yet-initialized) state, not a mock. | 62  | A cleanup/archival rule enforced only by advisory prose or a hook that cannot detect "done"                                                                         | If a rule says "move X when done," the harness must (a) detect "done" from a signal the artifact actually carries and (b) fail a scan when the move is skipped — advisory echo is not enforcement. Make the done-signal machine-checkable, add a scan to `run-all-scans.mjs`, and align any hooks to the same signal. Incident: 13 completed `.agents/tasks/*.md` piled up unarchived because both task-tracking hooks decided "done" by grepping a `**Status**:` field the task-breakdown format never carries (so every file read `status: unknown`) and the Stop hook only echoed a reminder. Fix: `scripts/harness/check-task-archival.mjs` (done = `Status: completed` OR all checkboxes checked + `Spec:` in `spec-docs/done/`), wired into `harness:scan`; hooks now report `DONE, needs archival` (HARNESS-016, 2026-06-20).                                                                                                   |
| 63  | Relying on npm-style `pre`/`post<script>` lifecycle hooks under pnpm                                                                                                | pnpm disables pre/post scripts by default (`enable-pre-post-scripts=false`), so a `postbuild` never runs from `pnpm build`. Chain the step explicitly inside the script (`"build": "next build && <step>"`) or set the flag intentionally — and make the step's dependency a real `devDependency`, not an ambient binary. Incident: docs search returned 404 because the `postbuild` pagefind index step silently never ran and `pagefind` was not even a dependency (DOCS-002, 2026-06-27).                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 64  | Leaving Next.js `output: 'export'` internal links on default prefetch                                                                                               | A static export never emits the RSC `/<route>.txt?_rsc=…` payloads that `<Link>` prefetch fetches, so the deployed site logs console 404s. Incident: marketing + docs sites each logged 5–7 prefetch 404s (WEB-014/DOCS-002).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 65  | Gating color/motion on `process.env.NO_COLOR` truthiness                                                                                                            | The NO_COLOR convention disables when the variable is **present at any value, including empty**. | 70  | Replacing/unifying a contract by checking only that old symbols are gone (grep), or putting a function-valued field in a contract that crosses a transport boundary | When replacing or unifying a contract, inventory **every** capability/field of the contract(s) being replaced and prove each is preserved or consciously dropped with rationale — a presence/absence grep for old symbols is **not** proof that behavior survived, and silently dropping a security-sensitive field (e.g. masked/secret input → plaintext) is a regression. | 69  | Repeating a design-token value as literals across components                                                                                                        | Define the token once (CSS custom property) and reference it everywhere — `var(--token)` for solids, `color-mix(in srgb, var(--token) N%, transparent)` for alpha tints — so a palette change is a single edit. | 71  | Testing a multi-step interaction with a scripted user-input double but asserting only the final result                                                              | A scripted user-interaction double (one that replays answers and resolves `cancelled` once exhausted) lets a regression that DROPS an intermediate prompt — a confirmation, a wizard step — pass silently: the surplus scripted answer is simply never consumed and the outcome can still look right. Worked example: `packages/agent-command/src/provider/__tests__/provider-command-module.test.ts` (delete-confirm + setup-step request assertions over the `scripted-interaction.ts` double). Incident: the provider-wizard scripted tests would have passed even if the delete confirmation or a setup step were dropped (CMD-004 PR #886, caught by the Pre-Merge Code-Review Gate, 2026-06-29). |
