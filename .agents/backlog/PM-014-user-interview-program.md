## Background

데이터 없이 제품 결정을 내리는 것은 도박이다. 초기 10~100명의 목소리가 제품 방향을 결정한다. 현재 사용자 피드백 수집 체계가 없다.

## 작업 항목

- 인터뷰 참여자 모집 채널 구축 (GitHub Discussions 자원자, Discord)
- 인터뷰 가이드 작성 (30분 구성):
  1. 첫 인상: "처음 어떻게 발견했나? 첫 실행 경험은?"
  2. 포기 순간: "사용 중 그만둔 순간이 있었나?"
  3. 즐겨 쓰는 기능: "어떤 기능을 가장 많이 쓰나?"
  4. 불편한 점: "없어서 불편한 것은?"
  5. 추천 의향: "주변에 추천하겠나? 왜?"
- 월 1회 PM 보고서로 인사이트 정리 및 백로그 반영
- 인터뷰 보상: GitHub 기여자 크레딧, 얼리 액세스
