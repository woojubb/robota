
============================================================
📋 팀 협업 데모 (한국어)
============================================================

🎯 아키텍처:
사용자 명령 → Team 에이전트 → (필요시 작업 위임) → 팀 멤버들 → 최종 응답

📋 이 데모에서 보여줄 것:
• 단순한 작업은 팀 에이전트가 직접 처리
• 복잡한 작업은 전문화된 팀 멤버들에게 위임
• 성능 통계 및 분석

🚀 간소화된 API:
이 예제는 새로운 간소화된 createTeam API를 사용하며,
task_coordinator 템플릿이 최적화된 설정으로 팀 협업을 자동으로 처리합니다.

팀 응답:
React와 Vue.js의 주요 차이점 3가지는 다음과 같습니다:

1. **구조와 설계 철학**:
   - **React**: 컴포넌트 기반 아키텍처를 사용하며, 상태 관리와 UI 업데이트를 위해 Virtual DOM을 활용합니다. JSX 문법을 사용하여 HTML과 JavaScript를 결합합니다.
   - **Vue.js**: 템플릿 기반 아키텍처를 가지며, 직관적인 데이터 바인딩과 반응형 시스템을 제공합니다. HTML 템플릿을 사용하고, Vue 인스턴스를 통해 데이터와 메소드를 관리합니다.

2. **상태 관리**:
   - **React**: 상태 관리를 위해 Context API 또는 Redux와 같은 외부 상태 관리 라이브러리를 주로 사용합니다. 상태 관리는 컴포넌트 트리의 상위에서 하위로 전달됩니다.
   - **Vue.js**: Vuex라는 공식 상태 관리 라이브러리를 제공하며, Vue의 반응형 시스템을 기반으로 상태를 관리합니다. 데이터가 변경될 때 자동으로 UI가 업데이트됩니다.

3. **학습 곡선**:
   - **React**: JSX, 상태 관리, 생명주기 메소드 등 여러 개념을 익혀야 하므로 상대적으로 학습 곡선이 가파를 수 있습니다.
   - **Vue.js**: 더 직관적인 문법과 구조로 인해 초보자에게 더 쉽게 접근할 수 있으며, 기본적인 사용법을 배우는 것이 비교적 간단합니다.
