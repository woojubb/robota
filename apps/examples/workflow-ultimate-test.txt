
============================================================
📋 팀 협업 데모 (한국어)
============================================================

🎯 아키텍처:
사용자 명령 → Team 에이전트 → (필요시 작업 위임) → 팀 멤버들 → 최종 응답

📋 이 데모에서 보여줄 것:
• 단순한 작업은 팀 에이전트가 직접 처리
• 복잡한 작업은 전문화된 팀 멤버들에게 위임
• 성능 통계 및 분석

🚀 간소화된 API:
이 예제는 새로운 간소화된 createTeam API를 사용하며,
task_coordinator 템플릿이 최적화된 설정으로 팀 협업을 자동으로 처리합니다.

팀 응답:
React와 Vue.js의 주요 차이점 3가지는 다음과 같습니다:

1. **구성 요소 및 데이터 바인딩**:
   - **React**: 컴포넌트 기반 아키텍처를 사용하며, JSX 문법을 통해 HTML과 JavaScript를 결합합니다. 데이터 바인딩은 단방향으로, 부모 컴포넌트에서 자식 컴포넌트로만 흐릅니다.
   - **Vue.js**: 템플릿 기반 접근 방식을 사용하며, HTML 템플릿을 사용하여 UI를 정의합니다. 양방향 데이터 바인딩을 지원하여, 데이터의 변화가 UI에 즉시 반영되고, UI의 변화가 데이터에 즉시 반영됩니다.

2. **상태 관리**:
   - **React**: 상태 관리를 위해 Redux, MobX와 같은 외부 라이브러리를 주로 사용합니다. 상태 관리가 복잡해질 수 있으며, 이를 위해 추가적인 설정이 필요합니다.
   - **Vue.js**: Vuex라는 내장 상태 관리 라이브러리를 제공하여, 상태 관리를 보다 쉽게 할 수 있습니다. Vuex는 Vue의 반응형 시스템과 잘 통합되어 있습니다.

3. **생태계 및 커뮤니티**:
   - **React**: Facebook에서 개발 및 유지보수하며, 큰 커뮤니티와 풍부한 생태계를 가지고 있습니다. 다양한 서드파티 라이브러리와 도구들이 존재합니다.
   - **Vue.js**: Evan You가 개발했으며, 비교적 작은 커뮤니티지만, 빠르게 성장하고 있습니다. Vue는 공식적으로 제공되는 플러그인과 도구들이 잘 정리되어 있어 사용하기 쉽습니다.
