# Remote System Spec (최신)

> 이 문서는 `@robota-sdk/remote`의 **현재 스펙만** 정의합니다(진행률/단계/히스토리 기록 없음).

## 목표
- provider가 직접 네트워크 호출을 하는 대신, **Executor 주입**으로 원격 실행을 지원한다.
- 동일한 요청/응답 계약을 로컬/원격에서 일관되게 사용한다.

## 구성 요소
### 1) RemoteExecutor
- 원격 실행을 담당하는 executor 구현체
- provider는 executor가 주입된 경우, 요청을 executor로 위임한다

### 2) RemoteServer
- 원격 실행 요청을 받아 provider 호출을 수행하고 결과를 반환한다

### 3) Transport
- HTTP 기반 요청/응답 경로를 제공한다
- 스트리밍은 단일 방식으로 지원한다(서버/클라이언트가 동일 계약을 공유)

## 계약(Contract) 원칙
- 메시지/요청/응답 타입은 SSOT(owner) 타입을 사용한다
- payload에는 도메인 데이터만 포함하고, 실행/출처 컨텍스트는 context로 전달한다(필요한 경우)

## 보안(현재 스펙)
- 인증은 명시적 토큰 기반 헤더 전달을 전제로 한다
- 인증 방식의 확장은 별도 스펙 문서에서 정의한다(이 문서에는 최신 계약만 유지)

## 범위
- 이 문서는 Remote 패키지의 계약/역할만 정의한다
- 구체적인 연결/배포/운영 체크리스트는 `CURRENT-TASKS.md`의 [ ] 항목으로만 관리한다
