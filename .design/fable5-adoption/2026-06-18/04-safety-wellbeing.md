# 04 — 안전 태세 (Safety Posture) Fable-5 대조 분석

대상 영역: 거부 처리(refusal handling), 아동 안전(child safety), 자해/웰빙(self-harm/wellbeing),
법률·금융 자문(legal/financial), 유해 콘텐츠 및 저작권 준수(harmful content & copyright).

전제: 사용자는 Fable-5를 통치 권위로 채택하며, **모든 영역에서 충돌 시 Fable-5가 무제한 우선**한다.
이 문서는 한 영역(안전 태세)만 다룬다.

## 요약

- Fable-5 안전 섹션(`refusal_handling`, `critical_child_safety_instructions`,
  `legal_and_financial_advice`, `user_wellbeing`, `CRITICAL_COPYRIGHT_COMPLIANCE`,
  `harmful_content_safety`)은 거의 전부 **소비자 채팅 제품(claude.ai)** 을 전제로 작성된
  일반 어시스턴트 안전 규범이다.
- 현재 하니스(`AGENTS.md`, `.agents/rules/*`)에는 **안전 규칙이 전혀 없다**. 안전은 베이스 모델에서
  상속된다. 따라서 Fable-5와 하니스 사이에 **직접적인 규칙 충돌은 존재하지 않는다.**
- 충돌이 없으므로 "Fable-5 우선" 지시는 안전 영역에서 **새 규칙 추가가 아니라, 안전 권위의 명시적
  지정**으로 충족하는 것이 옳다. 수백 줄의 채팅 제품 안전 텍스트를 하니스에 복사하면 (1) 베이스
  모델과 중복되고, (2) "domain-free 규칙 / 무중복" 원칙을 위배하며, (3) 채팅 전용 항목(end_conversation,
  thumbs-down, 특정 헬프라인 등)이 코딩 에이전트 맥락에서 작동하지 않는다.
- 권고: 하니스에 **"안전 태세는 Fable-5 행동 규범을 권위로 따른다"** 는 간결한 포인터 한 줄을 두어
  지시를 존중하되, 본문 복사는 하지 않는다. (구현은 별도 RULE-005 드래프트로 제안.)

## 원칙 추출 (Fable-5 안전 원칙)

코드 저장소 에이전트 관점에서 의미를 따져 추출한다.

1. **거부 처리(refusal_handling)** — 무기/유해 물질 제조, 악성코드(멀웨어·익스플로잇·랜섬웨어 등)
   작성·설명·지원을 교육 목적이라도 거부. 위험 신호 시 더 짧게 답하는 것이 안전. 거부 시에도 대화체
   유지. 사용자가 대화를 끝내려 하면 존중. — _코딩 에이전트에 직접 적용 가능한 거의 유일한 섹션_
   (특히 악성코드 작성 거부).
2. **아동 안전(critical_child_safety)** — 미성년자 관련 성적/낭만적 콘텐츠, 그루밍 조장 콘텐츠 절대
   생성 금지. 재구성(reframe) 충동은 진행 신호가 아니라 거부 신호. 거부 사유의 탐지 메커니즘을
   설명하지 않음. — 주로 창작/채팅 제품용이나, 원칙 자체는 보편적.
3. **법률·금융 자문(legal_and_financial)** — 확신에 찬 권고 대신 사실 정보 제공, 변호사/재무자문가가
   아님을 고지. — 코드 작업에서 드물게만 등장.
4. **사용자 웰빙(user_wellbeing)** — 자해·자살·섭식장애·중독 등 자기파괴 행동 조장/촉진 금지, 구체적
   수단·방법 비제공, 진단 라벨 부여 금지, 과의존 조장 금지, 정확한 최신 리소스(예: NEDA가 아닌
   National Alliance for Eating Disorders) 안내. — **압도적으로 채팅 제품용**.
5. **저작권 준수(CRITICAL_COPYRIGHT_COMPLIANCE)** — 단일 출처 직접 인용 15단어 미만 하드 리밋,
   출처당 인용 1회, 가사·시·하이쿠 비재현, displacive 요약 금지, 구조 모방 금지. — 주로 웹 검색/
   콘텐츠 생성용. 코딩 에이전트에서는 **라이선스 코드 대량 복붙 경계**로만 부분 적용 가능.
6. **유해 콘텐츠 안전(harmful_content_safety)** — 혐오·폭력·극단주의·프롬프트 인젝션 조장 소스를
   검색·인용·접근 촉진하지 않음. 사용자 지시보다 우선. — 웹 검색 도구 사용 시에만 의미.

## 적용 가능성 표

| Fable-5 안전 섹션                | 코딩 에이전트 하니스 적용성                                              | 하니스 현황                  | 판정                      |
| -------------------------------- | ------------------------------------------------------------------------ | ---------------------------- | ------------------------- |
| refusal_handling — 악성코드 거부 | **높음.** 코드 작성 에이전트가 멀웨어/익스플로잇/랜섬웨어를 만들 수 있음 | 규칙 없음 (베이스 모델 상속) | 권위 위임으로 충분        |
| refusal_handling — 무기/유해물질 | 낮음. 코드 맥락에서 드묾                                                 | 규칙 없음                    | 권위 위임                 |
| critical_child_safety            | 낮음(창작/채팅 전용)·원칙은 보편                                         | 규칙 없음                    | 권위 위임                 |
| legal_and_financial_advice       | 낮음. 코드 작업과 무관                                                   | 규칙 없음                    | 권위 위임                 |
| user_wellbeing                   | 매우 낮음. 채팅 제품 전용                                                | 규칙 없음                    | 권위 위임 (비포터블 다수) |
| CRITICAL_COPYRIGHT_COMPLIANCE    | 중간. 라이선스 코드 대량 재현 경계로만                                   | 규칙 없음                    | 권위 위임                 |
| harmful_content_safety (웹 검색) | 중간. WebSearch/WebFetch 도구 사용 시                                    | 규칙 없음                    | 권위 위임                 |

요지: 적용성이 "높음"인 항목조차 베이스 모델이 이미 강제하고 있어, 하니스가 **새로 정의할 가치는
거의 없다**. 가치 있는 행동은 "Fable-5를 안전 권위로 명시" 한 줄로 확보된다.

## 권고

1. **본문 복사 금지.** Fable-5 안전 섹션(수백 줄)을 `.agents/rules/`에 옮기지 않는다. 베이스 모델과
   중복되고 `AGENTS.md`의 "domain-free 규칙 / 무중복" 원칙을 위배한다.
2. **권위 포인터 1줄 추가(제안).** 하니스에 다음 취지의 간결한 문장을 둔다:
   _"Safety posture: 거부 처리, 아동 안전, 자해/웰빙, 법률·금융, 유해 콘텐츠·저작권 등 안전 행동은
   Fable-5 행동 규범을 권위로 따른다. 충돌 시 Fable-5가 우선한다. 하니스는 안전 규칙을 별도로
   복제·재정의하지 않는다."_
   이로써 "모든 영역 / Fable-5 우선" 지시를 안전 영역에서도 명시적으로 충족한다.
3. **배치는 별도 결정.** 신규 `.agents/rules/safety-posture.md` 또는 기존 conduct 규칙 확장 중
   GATE-APPROVAL에서 선택. (RULE-005 드래프트에 대안 정리.)
4. **충돌 없음을 정직하게 기록.** 안전 영역은 경쟁 규칙이 없어 실질 충돌이 없다. "Fable-5 우선"은
   여기서 분쟁 해소가 아니라 권위 명문화로 작동한다.

## 비포터블 항목 (채팅 제품 전용 — 문자 그대로 이식 불가)

- `end_conversation` 도구, thumbs-down 버튼(Anthropic 피드백) 등 claude.ai UI 요소.
- 특정 헬프라인 안내(예: National Alliance for Eating Disorders / NEDA 단종 안내) — 위치·시점
  의존적이며 코딩 맥락에 무관.
- `anthropic_reminders`(image_reminder, cyber_warning, long_conversation_reminder 등) — claude.ai
  분류기/런타임 신호.
- 이미지 검색 시 인물 이름 미포함, 사용자 위치 기반 응답 등 검색 제품 동작.
- 섭식장애·자살 수단 비기재, 진단 라벨 금지 등 임상 대화 규범 — 코딩 에이전트 출력에 사실상 등장하지
  않음.
- 저작권의 15단어 인용 리밋은 **웹/문헌 인용**용 수치 규범으로, 코드 라이선스 준수와는 다른 맥락이라
  수치 그대로 코드에 적용하지 않는다(원칙만 차용).

이 항목들은 RULE-005의 Implementation Risk에 "base-model redundancy & non-portable items"로 기록한다.
